## 1. 배경 및 문제

**차지(CHAGEE)** 매장 점장이 매주 직원 10명 내외의 근무표를 엑셀로 수동 작성 중.

| 항목     | 현황                                          |
| -------- | --------------------------------------------- |
| 소요시간 | 약 5시간/회                                   |
| 도구     | Excel (수기 기록 참조)                        |
| 핵심문제 | 특정 직원이 마감/오픈에 반복 투입되는 걸 나중에야 발견 |
| 부수문제 | 공휴일·주말 휴무 배분의 누적 불균형           |

점장의 의도와 무관하게 발생하는 구조적 불공평 — 손으로 가중치를 추적하기에 변수가 너무 많다.

---

## 2. 목표

- 직원별 **누적 가중치(공평 지표)** 를 시스템이 추적
- 가중치 기반으로 **자동 스케줄 초안 생성**
- 점장이 검토 후 **확정/수정**하는 워크플로우

> "엑셀 5시간 → 자동 초안 확인 후 30분 이내 확정"이 성공 기준.

## 4. 핵심 개념

### 4.1 파트 유형

| 파트 | 시간대 (예시)   | 기본 부담 가중치      |
| ---- | --------------- | --------------------- |
| 오픈 | 08:00~14:00     | 1 (이른 출근)         |
| 미들 | 11:00~17:00     | 0 (선호도 높음)       |
| 마감 | 16:00~22:00     | 2 (늦은 퇴근)         |

### 4.2 휴일 유형

| 유형       | 보상 가중치 |
| ---------- | ----------- |
| 공휴일 휴무 | +3         |
| 주말 휴무  | +2          |
| 평일 휴무  | +1          |

### 4.3 직원 파트 성향 (신규)

각 직원마다 오픈/미들/마감에 대한 성향을 설정할 수 있다.

| 성향값 | 의미                     | 부담 가중치 보정 |
| ------ | ------------------------ | --------------- |
| `like` | 이 파트를 선호함         | ×0.5 (부담 감소) |
| `neutral` | 특별한 선호 없음      | ×1.0 (기본값)   |
| `dislike` | 이 파트를 기피함      | ×1.5 (부담 증가) |

**적용 방식:** 글로벌 부담 가중치에 개인 성향 계수를 곱해 해당 직원의 **개인화된 부담값**으로 계산.

예시: 마감을 좋아하는 직원이 마감 투입 → 부담값 = 2 × 0.5 = 1 (오픈과 동일)

### 4.4 공평 지표 (Fairness Score)

각 직원의 누적 점수:

```text
개인 부담값(파트, 직원) = 파트_기본_부담 × 성향_계수(파트, 직원)

fairness_score = Σ(개인 부담값) - Σ(보상 가중치)
```

점수가 **높을수록** 상대적으로 더 나쁜 조건을 소화한 것.
스케줄 생성 시 점수 높은 직원에게 좋은 파트/휴일을 우선 배정.

**예시 비교:**

| 직원   | 마감 성향 | 마감 5회 부담 | fairness_score |
| ------ | --------- | ------------- | -------------- |
| 박지호 | dislike   | 5×2×1.5 = 15  | 높음 (우선 보상 대상) |
| 한승우 | like      | 5×2×0.5 = 5   | 낮음 (상대적으로 OK) |

## 6. 목업 데이터 (MVP용)

### 직원 목록 (10명) + 파트 성향

| 이름   | 고용형태   | 근무 가능 요일       | 오픈  | 미들    | 마감    |
| ------ | ---------- | -------------------- | ----- | ------- | ------- |
| 김민준 | 풀타임     | 월~금                | neutral | like  | dislike |
| 이서연 | 파트타임   | 화~토                | dislike | neutral | neutral |
| 박지호 | 파트타임   | 월·수·금·토          | neutral | neutral | dislike |
| 최유나 | 풀타임     | 월~금                | like  | neutral | neutral |
| 정다은 | 파트타임   | 수~일                | neutral | like  | neutral |
| 한승우 | 파트타임   | 월·화·목·금          | neutral | neutral | like  |
| 오하늘 | 풀타임     | 월~토                | neutral | neutral | neutral |
| 윤지아 | 파트타임   | 화·목·토·일          | dislike | like  | neutral |
| 강도윤 | 파트타임   | 월~금                | like  | neutral | dislike |
| 임소희 | 풀타임     | 월~일                | neutral | neutral | neutral |

### 과거 4주 이력 (목업)

- 마감 집중: 박지호, 한승우 (각 6~8회) — 박지호는 dislike라 점수 높음, 한승우는 like라 점수 낮음
- 오픈 집중: 이서연, 윤지아 (각 5~7회) — 둘 다 dislike라 점수 높음
- 공휴일 휴무 편중: 김민준, 최유나 (각 3회)
- 균형 양호: 임소희, 오하늘

## 7. 알고리즘 (Greedy Fairness Scheduler)

```text
1. 대상 주의 각 날짜 × 파트 슬롯 목록 생성
2. 각 직원의 현재 fairness_score 계산 (성향 반영)
3. 슬롯마다:
   a. 해당 날 출근 가능한 직원 필터
   b. 이미 당일 배정된 직원 제외
   c. fairness_score 높은 순 정렬
   d. 상위 후보 중 랜덤 배정 (동점 처리)
4. 휴무 배정:
   a. 잔여 인원 중 공휴일/주말 휴무를 fairness_score 역순으로 배정
5. 결과를 초안으로 저장
6. (선택) 성향 매칭률 지표 출력:
   "선호 파트 배정 비율: N%"
```

## 10. 성공 기준 (MVP)

- [ ] 10명 직원 데이터 등록 상태에서 1주 스케줄 자동 생성 < 3초
- [ ] 생성된 스케줄의 마감/오픈 편차가 시뮬레이션 4주 후 ±1회 이내
- [ ] 성향 반영 시 `like` 파트 배정 비율이 `dislike` 파트 배정 비율보다 높음
- [ ] 점장이 초안 검토 후 수정 없이 또는 최소 수정으로 확정 가능
- [ ] 시연 시 "공평하게 짜여졌다"고 납득 가능한 근거 화면 존재
